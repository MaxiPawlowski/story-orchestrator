# Gotchas

- **`npm run lint` enumerates src dirs explicitly** (see package.json) — a new top-level `src/` dir is silently unlinted until added to the script. Same for tsconfig `include`.
- **In-page debug handle**: `globalThis.storyOrchestratorRuntime` (RuntimeManager) — `importStory(json)`, `runExtractionNow(response?, reason?)`, `getSnapshot()`. Plan 07 adds: `detectSceneBreak()`, `runSceneBreakPass(audit)`, `getEnabledCharacterIds()`, `runMemorizeBacklog(windowSize?)`, `setMemoryPinned(id, bool)`, `excludeMemoryEntry(id)`, `editMemoryEntry(id, text)`. Plan 08 adds: `runConsolidation()` (P4 dedup + supersession + bridge + WI sync), `runSupersessionBridge(entries)`, `syncWorldInfo()`, `getMemoryInjectionBlocks()`. Plan 09 adds: `getArcs()`, `getOpenArcs()`, `setArcPinned(id, bool)`, `removeArc(id)`, `runArcSummaryPass(arcIds)` (P4: resolved-arc summaries → canon regen), `getCanon()` (derived canon, falls back to canon-lite), `regenerateCanon(force?)`. Arc `[arc]`/`[resolved]` lines ride the same shared read as MEMORY/DELTA; a confirmed `[resolved]` matching an authored `arc_bridges` keyword enqueues a code-side `progress_toward_<anchor>` increment at the next boundary. Plan 10 adds: `getEpistemic()`, `getLedger()` (LedgerView[] incl. blackboard-mirrored bound rows), `getEpistemicBlock()`/`getLedgerBlock()` (macro/injection text), `runEpistemicLedgerPass(audit)` (P2 combined per-scene epistemic + ledger pass, `[retire]` supersession), `onMemberDrafted(chId)`/`clearPrivateInjection()` (private per-draft swap + restore), `setEpistemicLedgerCapable(bool)`, `setEpistemicPinned(id,bool)`/`removeEpistemicEntry(id)`/`setLedgerPinned(id,bool)`/`removeLedgerEntry(id)`. Epistemic `[knows]/[unaware]/[suspects]/[believes]/[hiding]` + ledger `[state:Entity:type] field=value` lines ride the same shared read as MEMORY/DELTA **only when the capability profile is on**; the full pass runs at scene breaks. A ledger field bound to a quality via `ledger_binding {entity, field}` is written **only** through the blackboard (single writer) and mirrored read-only into `getLedger()`; extracted `[state:…]` lines for a bound field are dropped. Epistemic is **never** written to World Info (privacy). Debug-response globals: `storyOrchestratorDebugExtractionResponse` (shared read), `storyOrchestratorDebugSceneSummaryResponse` (P2 scene-summary pass), `storyOrchestratorDebugSupersessionResponse` (plan 08 latching-bridge DELTA pass), `storyOrchestratorDebugArcSummaryResponse` (plan 09 arc-resolution summary), `storyOrchestratorDebugCanonResponse` (plan 09 canon synthesis), `storyOrchestratorDebugEpistemicResponse` + `storyOrchestratorDebugLedgerResponse` (plan 10 P2 pass). Used by gate check scripts and ad-hoc `browser_evaluate`.
- **Plan 08 embeddings use ST's vectors backend** (`/api/vector/{insert,query,purge}` via `stHost/vectors.ts`, `source: 'transformers'`, no config). Query returns threshold-filtered `metadata` (score dropped server-side at `src/endpoints/vectors.js:390`) — dedup bands are read via two threshold queries (dup 0.82 / same-topic 0.55), not raw scores. Jaccard (`similarity.ts`) is the fallback when no source. Consolidation runs P4 on a boundary cadence and only for tier groups ≥ `CONSOLIDATION_MIN_GROUP` (8).
- **`so-state` reads persisted `chat_metadata`** — live in-page values between persist cycles may differ.
- **Settings panel root keeps legacy id `#stepthink_settings`** — debug selectors depend on it.
- **Jest runs `--runInBand`**; tests colocated as `src/**/*.test.ts`, excluded from tsconfig/lint.
- **`global.d.ts`** imports ST type declarations via relative paths into the ST repo (`../../../../public/global`) — resolves only inside the ST tree.
- **Tailwind 4** uses `@tailwindcss/postcss` plugin (not classic `tailwindcss` PostCSS plugin).
- **Webpack fallbacks** disable `fs`, `http`, `https`, `url`, `crypto` — no node builtins in browser target.
- **`dist/` is gitignored**; run `npm run build` before committing only if dist needs updating.
